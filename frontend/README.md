# Nexus AI Frontend

The frontend for Nexus AI is a modern, high-performance web application built with **Next.js 16 (App Router)** and **TypeScript**. It is designed to be mobile-first and deeply responsive, providing a premium, immersive chat experience that mirrors the fluid intelligence of Nexus AI.

---

## 🎨 Design System & Aesthetics

Nexus AI's UI follows a **Glassmorphic** and **Modern Dark** aesthetic:
- **Vibrant Gradients:** Harmonious HSL-tailored colors used for Nexus AI's "evolving orb."
- **Animated Orbs:** The interface feels alive with micro-animations and orbital pulses representing the backend's cognitive load.
- **Glassmorphism:** Frosted-glass components (`bg-glass`) for an airy, premium feel.

---

## 🏗 Technology Stack

- **Framework:** Next.js 16 (App Router)
- **State Management:** React Hooks (Context-less optimization for speed)
- **Styling:** Vanilla CSS Modules (Maximum flexibility and zero runtime overhead)
- **Authentication:** Supabase Auth (Supporting Google OAuth & Email/Password)
- **AI Rendering:** `react-markdown` for beautiful bold, code, and list formatting.
- **Real-time:** Server-Sent Events (SSE) for fluid, token-by-token message streaming.

## 🚀 Quick Start

### Prerequisites
- Node.js 20+
- `npm` or `pnpm`

### 1. Installation

```bash
cd frontend
npm install
```

### 2. Environment Configuration

Copy the example environment file:

```bash
cp .env.example .env.local
```

Your `.env.local` should contain:

```ini
NEXT_PUBLIC_SUPABASE_URL="your-supabase-url"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your-anon-key"
NEXT_PUBLIC_API_URL="http://localhost:8000"
```

### 3. Running Development Server

```bash
npm run dev
```

Visit `http://localhost:3000`.

## 🧬 Key Features

### 1. Token Streaming
Uses custom SSE (Server-Sent Events) listeners in `chat/page.tsx` to render tokens the moment they are generated by the backend, ensuring zero perceived latency for the user.

### 2. Evolution Dashboard
A visual playground in `evolution/page.tsx` that displays System's `empathy_depth`, `knowledge_breadth`, and `wisdom_score` using complex CSS animations and CSS-variable driven progress bars.

### 3. Identity-Aware Theming
The UI colors and System's Orb behavior change based on System's current evolution stage (`Nascent` → `Growing` → `Mature` → `Transcendent`).
